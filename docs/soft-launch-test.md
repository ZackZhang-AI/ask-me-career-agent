# Ask Me 五人软上线测试

本表用于上线后的真实用户验证，目前是待执行模板，不代表已经完成真人测试。

## 测试对象与统一任务

招募 5 名与目标岗位相关且未参与项目开发的人：2 名 AI 产品招聘经理或业务面试官、2 名专业 HR、1 名 AI 产品从业者。若无法匹配同等角色，可以同类招聘决策者替换，但需记录实际身份。

每人使用同一 Production 地址，独立完成以下 60 秒任务：

1. 浏览首屏并说出候选人的 3 个核心差异点。
2. 提一个简历之外的能力问题，再追问一次“这个项目中他具体做了什么”。
3. 打开至少一个来源或项目链接，判断证据是否支持回答。
4. 选择是否愿意打开简历或联系候选人，并说明原因。

主持人只读任务，不讲解页面、不引导点击；测试结束后再追问“哪些内容增加信任、哪些内容降低信任”。不得记录参与者姓名、邮箱、电话或公司机密。

## 通过标准与处理规则

- 至少 4/5 人能复述 3 个核心差异点。
- 至少 3/5 人明确表示获得了简历之外的增量信息。
- 5/5 人可以找到项目证据，且没有把证据边界误解为已验证业绩。
- 严重事实错误、隐私泄露或机密输出必须为 0；出现任一项即停止软上线。
- 汇总只修复阻碍理解、信任和转化的问题。每个问题记录“现象、影响人数、证据、修改方案、复测结果”，不在此阶段增加 JD 评分、多语言或分享等 P1 功能。

完成五人测试后，把聚合结果写入同一文档的新日期章节；原始记录保留匿名编号，不补填或推测缺失结果。
