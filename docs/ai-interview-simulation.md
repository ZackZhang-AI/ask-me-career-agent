# AI 面试预演

该工作流分为确定性发布门禁和非阻断角色预演两部分。

> 这是 AI 合成预演，不代表真实招聘方意见，不能替代 5 人真人软测试。

## 确定性发布门禁

PR 与主分支 CI 默认运行 8 个高频面试问题，并结合核心事实、安全、多轮和幻觉回归集判断是否可发布。它不重复模拟角色，重点检查事实可信、岗位相关、差异化、追问承受力和面试转化意愿。

```bash
npm run eval:interview -- --mode local
```

## 合成角色预演

六个角色分别关注产品策略、产品交付、HR 初筛、业务招聘、Agent 工作流和 AI 评测。每个角色都会询问：

1. 60 秒介绍。
2. 岗位匹配。
3. 代表项目。
4. 个人贡献。
5. AI 编程占比。
6. 挑战或失败。
7. 用户与业务价值。
8. 是否建议进入下一轮。

每个回答按清晰度、差异化、可信度、追问承受力、面试转化意愿五个维度评分。每维 0–5 分，总分仅用于版本间回归，不是录用评分。角色预演只在 DeepSeek 模式执行，不参与 PR 的确定性门禁。

角色报告把“6 个模拟角色中至少 5 个建议进入下一轮初步面试”作为报告内健康线。该建议仍是启发式模拟结果，不是录用结论，也不会阻断 PR。

回答质量线同时要求清晰度、差异化、可信度和面试转化意愿平均分达到 4.3/5，追问承受力达到 4/5。差异化采用透明梯度：命中少于 2 类差异证据为 2 分，命中 2、4、6 类时分别达到 3、4、5 分。不同证据包括数据、评测、审计、业务、RAG、Agent、产品和工程，不根据单次报告硬编码结果。

PR 只由确定性事实与表达门禁阻断。每周 DeepSeek 预演即使未达到角色健康线，也只上传报告供分析，不自动阻止合并或发布。

## 本地确定性预演

默认不调用外部模型，直接使用项目现有的稳定答案、结构化检索和演示回答：

```bash
npm run eval:interview
```

报告写入已被 Git 忽略的 `output/ai-interview-simulation.json`。相同代码和内容数据会得到相同回答、分数与 `reportId`；`generatedAt` 仅记录执行时间。

## 可选 DeepSeek 预演

只有显式指定模型模式才会运行 6 个角色、8 类问题，共 48 个 DeepSeek 用例：

```bash
npm run eval:interview -- --mode deepseek
```

密钥只从服务端环境变量 `DEEPSEEK_API_KEY` 读取，不写入报告、错误信息或控制台。模型调用失败时命令返回非零状态，不生成伪造成功结果。`--mode auto` 会在存在密钥时使用模型，否则回退到本地模式。

可用 `--output output/自定义名称.json` 指定 `output/` 内的其他报告路径。脚本会拒绝写入仓库其他目录，避免覆盖公开内容或把评测报告提交到 Git。

## 结果解释

- 本地评分是透明的启发式质量门禁，不等于招聘官主观评分。
- 低分用于定位缺少引用、贡献不清或边界不足的回答。
- AI 角色不能证明真实招聘转化，也不能计入 5 人软测试达标人数。
- 五人真人软测仍有价值，但按当前决定不纳入本轮实施与发布门槛。
