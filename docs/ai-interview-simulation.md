# AI 面试预演

该工作流用 6 个合成角色、8 类问题对 Ask Me 的回答质量做发布前回归，共 48 个用例。

> 这是 AI 合成预演，不代表真实招聘方意见，不能替代 5 人真人软测试。

## 覆盖范围

六个角色分别关注产品策略、产品交付、HR 初筛、业务招聘、Agent 工作流和 AI 评测。每个角色都会询问：

1. 60 秒介绍。
2. 岗位匹配。
3. 代表项目。
4. 个人贡献。
5. AI 编程占比。
6. 挑战或失败。
7. 用户与业务价值。
8. 是否建议进入下一轮。

每个回答按清晰度、差异化、可信度、追问承受力、面试转化意愿五个维度评分。每维 0–5 分，总分仅用于版本间回归，不是录用评分。

最终门禁不是单题通过率，而是 6 个模拟角色中至少 5 个基于全部八类回答建议进入下一轮初步面试。该建议仍是启发式模拟结果，不是录用结论。

同时设置回答质量门禁：清晰度与差异化的 48 题平均分都必须达到 4/5。差异化采用透明梯度：命中少于 2 类差异证据为 2 分，命中 2、4、6 类时分别达到 3、4、5 分。不同证据包括数据、评测、审计、业务、RAG、Agent、产品和工程，不根据单次报告硬编码结果。

只有“至少 5/6 个角色建议进入下一轮”和“清晰度、差异化平均分均达到 4”同时成立，最终上线门禁才通过。CLI 在任一门禁未通过时返回非零退出码，便于 CI 阻止发布。

## 本地确定性预演

默认不调用外部模型，直接使用项目现有的稳定答案、结构化检索和演示回答：

```bash
npm run eval:interview
```

报告写入已被 Git 忽略的 `output/ai-interview-simulation.json`。相同代码和内容数据会得到相同回答、分数与 `reportId`；`generatedAt` 仅记录执行时间。

## 可选 DeepSeek 预演

只有显式指定模型模式才会调用 DeepSeek：

```bash
npm run eval:interview -- --mode deepseek
```

密钥只从服务端环境变量 `DEEPSEEK_API_KEY` 读取，不写入报告、错误信息或控制台。模型调用失败时命令返回非零状态，不生成伪造成功结果。`--mode auto` 会在存在密钥时使用模型，否则回退到本地模式。

可用 `--output output/自定义名称.json` 指定 `output/` 内的其他报告路径。脚本会拒绝写入仓库其他目录，避免覆盖公开内容或把评测报告提交到 Git。

## 结果解释

- 本地评分是透明的启发式质量门禁，不等于招聘官主观评分。
- 低分用于定位缺少引用、贡献不清或边界不足的回答。
- AI 角色不能证明真实招聘转化，也不能计入 5 人软测试达标人数。
- 正式上线仍需真人完成 60 秒理解、证据查找、简历增量信息和面试意愿验证。
